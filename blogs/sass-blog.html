Is Sass A Good Substitute For CSS?

Aim:

The aim of this project is to broaden my knowledge (the little amount of it) on the CSS extention, Sass. I aim to learn and use Sass code correctly and in an efficient manor by using a Sass compiler, such as Koala, to convert my Sass code into CSS.



Objectives

My main objectives are:

  • To learn about Sass and why using it is a suitable substitute for CSS
  • To learn about Sass compilers and why they are a useful tool to use alongside Sass
  • To use Sass in my code to make my code more efficient



Definition:

I will use the CSS extension Sass to learn the code to see if using Sass to develop the design of my website. To do this I will be looking at Sass’ nesting features, functions, and media query includes. As well as applying variables for fonts and colours that may be used many times throughout my code.

To help me compile my Sass code (and all other code) I will be using a compiler application: Koala.



Set Up:

Before I could start coding in Sass, I needed to install it to my computer by using Cmder. A command line emulator which I would use to install the latest version of Sass as a Node.js package by entering npm install –g sass.

Once Sass was installed and the Sass file was created in my project folder, I had to add the Sass file to my compiler, Koala.

Adding the folder to Koala so that it can automatically compile the sass file into CSS

Koala’s user interface shows that the main.scss is highlighted green, as this is the file that is being compiled. The main.css file is greyed out as this is the file where all the Sass code will be outputted.

This alert informs us that the save and compile of the Sass file was a success.

How:

To demonstrate how I learned and used Sass, I decided to create a basic landing page with some minimal content as to focus more on coding the Sass.

One of the great things about using Sass is the ability to use variables.

“Think of variables as a way to store information that you want to reuse throughout your stylesheet. You can store things like colors, font stacks, or any CSS value you think you'll want to reuse.” (Sass-lang.com, n.d)

Variables allow you to define a value for a property, such as the main body font size, and call upon it multiple times within the code. The useful thing about Sass variables is, if the size of the font needs to change and the code has been written many times, just by changing the value of the variable, will update the new value for wherever the variable is called in the code.

This shows some variables being defined and how you reference them as a value.

How it looks in CSS

Media queries are a vital in modern web development. Allowing a website to accommodate all device sizes from desktop to mobile is a curtail part of making a website accessible.

“Media queries let you adapt your site or app depending on the presence or value of various device characteristics and parameters… they are a key component of responsive design.” (MND Web Docs, 2019)



Sass reduces the tedious code writing that comes with writing media queries. To implement media queries you can define the minimum width, like you normally would e.g. min-width: 840px;, as a variable which you then feed into mixin function.

By using this mixin function, you can add an include to the section of the site that would need editing on a smaller screen (or larger when doing a mobile first approach). Turning this:

“Nesting in Sass allows you to write descendant selectors in a syntax that is more natural and easier to read.” (Belman, 2016)

Nesting is a very useful feature that Sass has over CSS. Nesting is a clean way of writing code were the parent and child properties values need to be defined together. Evaluation:

After using Sass as appose to CSS I can now answer my question that I asked myself; is Sass a good substitute for CSS?

And my answer is no. I think Sass is a good partner to CSS. Using Sass to write your code and compiling into CSS is a good method to writing code. The readability and features that Sass has over CSS is a big benefit. However, being able to automatically compile the Sass code into CSS with a compiler, such as Koala, is the way forward with modern front-end development.
